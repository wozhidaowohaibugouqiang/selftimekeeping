## 2. 当前实现的主要功能

### 2.1 守时主流程
- 使用 `gps_pps` 与本地 `dpps` 做周期差值测量（`DIFF`）。
- 支持稳定等待阶段（前置多个窗口不训练）。
- 稳定后进入参数训练，锁定固定补偿参数 `CA`。
- 在线补偿输出 `CS`（步进值，含余数均匀分配）。
- 动态补偿就绪标志 `DR` 用于控制是否启用补偿。

### 2.2 补偿参数机制
- `CA`：固定残差平均值（训练得到，锁定后稳定）。
- `CS`：每秒补偿步进（由 `CA` 分摊得到，常见在相邻两个值间抖动）。
- `DY`：动态补偿偏移（当前版本按“每秒直接应用步进，不跨秒线性累加漂移”的策略运行）。

### 2.3 Flash 记录与回读
- 支持差值数据写入 Flash。
- 支持 dump 导出：`DMP_META`（FWE/FRE/LIMIT）+ `ADDR/DATA`。
- 支持回读验证，`FL_DONE/WD/LW` 可观测。

### 2.4 串口调试观测
- UART 周期输出关键字段：
  - `DIFF/FWE/FRE/FL_DONE/WD/LW`
  - `DY/CS/CV/DR/CA`
  - `LCMD/RX1/JV/JI`
- 便于在线判断训练、补偿、存储三条链路是否一致。

## 3. 各模块功能（简版）

- `top.v`：当前主顶层，状态机与模块集成、补偿注入与调试信号汇总。
- `top_enhanced.v`：增强版顶层（保留）。
- `timekeeping_core.v`：产生本地 dpps，计算与 gps pps 的差值。
- `avg_remain_comp.v`：均值/余数分配相关补偿处理。
- `dynamic_comp_coeff.v`：动态补偿系数链路。
- `aging_comp_coeff.v`：老化项补偿系数链路。
- `pid_dynamic_comp.v`：动态补偿控制（PID相关）。
- `sec_tick.v`：秒节拍生成。
- `long_period_counter.v`：长周期统计/计数。
- `osc_stabilizer.v`：上电稳定等待。
- `param_manager.v`：参数组织与管理。
- `fifo_write_module.v`：差值写 FIFO。
- `fifo_read_module.v`：数据从 FIFO/Flash 读出链路。
- `flash_controller.v`：基础 Flash 控制。
- `flash_controller_enhanced.v`：增强 Flash 控制与调试。
- `spi_driver.v`：SPI 读写时序驱动。
- `uart.v`：串口收发与调试报文打包。

## 4. 最新调试近况（基于 0402 串口日志）

### 4.1 已确认正常
1. 训练前阶段：`DR=0` 时 `CA=0/CS=0`，未提前训练补偿。
2. 训练后阶段：`DR=1` 后 `CA` 锁定为稳定负值（示例 `-000009AD`）。
3. `CS` 在 `-009A/-009B` 间抖动，符合分摊逻辑预期。
4. `DIFF` 从约 `-0x9B0` 逐步收敛到 `-3~+3`，长期围绕 0 小幅波动。
5. Flash dump 元信息一致（示例 `FWE=FRE=LIMIT`），读写链路闭环正常。

### 4.2 仍需注意（非阻塞）
- dump 切换瞬间偶有单点异常值（边界样本），但不影响稳态闭环。
- 若后续用于正式交付，建议加“dump退出后丢弃首个统计窗口”的保护逻辑（可选）。
