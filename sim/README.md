# 自主守时系统仿真说明

## 仿真文件结构

```
sim/
├── tb_timekeeping_core.v       # 测试文件（自动测试三种场景）
├── timekeeping_core.v          # 实现文件（自主守时核心模块）
└── README.md                   # 本说明文件
```

## 系统功能

本仿真系统实现了一个自主守时系统的核心功能，主要包括：

1. **GPS PPS 同步**：接收并同步GPS标准PPS信号
2. **动态偏移DPPS生成**：根据输入的偏移量生成带偏差的本地DPPS信号
3. **时间偏差计算**：计算DPPS与GPS PPS之间的时间偏差
4. **阈值判断**：判断偏差是否超过设定阈值

## 仿真场景

测试文件 `tb_timekeeping_core.v` 自动测试三种场景，按时间顺序切换：

| 场景 | 时间范围 | DPPS状态 | 偏移量设置 | 预期结果 |
|------|---------|---------|-----------|---------|
| 1    | 0~2秒   | 与GPS PPS对齐 | offset=0 | 无偏差 |
| 2    | 2~4秒   | 超前GPS PPS | offset=2 (200ns) | 检测到超前偏差 |
| 3    | 4~6秒   | 滞后GPS PPS | offset=3 (300ns) | 检测到滞后偏差 |

## 信号说明

### 输入信号
- `sys_clk_10mhz`：10MHz晶振时钟，周期100ns
- `sys_rst_n`：系统复位，低有效
- `gps_pps`：GPS标准PPS信号（1秒1个脉冲）
- `dpps_offset`：DPPS动态偏移量输入（单位：100ns），由测试文件控制

### 输出信号
- `dpps`：带偏差的本地DPPS信号
- `diff_sign`：偏差符号（0=超前，1=滞后）
- `diff_data`：偏差值（单位：100ns）
- `diff_valid_flag`：偏差有效标志位（超过阈值时为1）

## 核心实现

### 1. GPS PPS同步消抖
通过两级寄存器同步GPS PPS信号，消除 metastability 问题。

### 2. 动态偏移DPPS生成
- 当 `dpps_offset=0` 时，DPPS与GPS PPS完全对齐
- 当 `dpps_offset>0` 时，可生成超前或滞后的DPPS信号：
  - 超前：计数到 `CYCLE_1S - dpps_offset` 时生成脉冲
  - 滞后：计数到 `CYCLE_1S + dpps_offset` 时生成脉冲

### 3. 差值计算
在GPS PPS上升沿时，根据当前DPPS计数器值计算偏差：
- 若计数器值小于半周期，视为超前
- 若计数器值大于半周期，视为滞后

### 4. 阈值判断
当偏差值大于等于设定阈值（100ns）时，置位有效标志。

## 仿真运行

1. 仿真时钟：10MHz（周期100ns）
2. 复位信号：初始低电平，200ns后拉高
3. GPS PPS：1秒1个脉冲，脉冲宽度100ns
4. 仿真时长：6秒（覆盖三种场景）
5. 仿真结束：6秒后自动停止

## 设计特点

1. **模块化设计**：核心逻辑与测试逻辑分离
2. **动态偏移支持**：通过输入参数控制偏移量，无需修改代码
3. **多场景自动测试**：单测试文件覆盖三种典型场景
4. **清晰的信号命名**：信号名称直观反映其功能
5. **详细的注释**：关键代码段配有详细注释说明

## 注意事项

- 本设计基于10MHz时钟，若使用其他频率时钟，需相应调整 `CYCLE_1S` 参数
- 偏移量单位为100ns（即1个时钟周期），实际应用中可根据需要调整
- 本仿真仅验证核心逻辑功能，实际应用中还需考虑噪声、温度等因素的影响
