# 自主守时系统

## 1. 项目概述

本项目实现了一个基于 EF2M45 FPGA 的高精度自主守时系统。系统在 GPS 信号正常时，实时测量并学习本地晶振相对于 GPS 的频率漂移特性；在 GPS 信号丢失时，自动切换至守时模式，利用历史学习数据进行频率补偿，维持本地秒脉冲 (DPPS) 的高精度输出。

## 3. 已实现功能模块说明

### 3.1 顶层模块 (`top.v`)

**功能说明**：
- 系统主控制器，整合所有子模块
- 实现系统主状态机，管理工作模式切换
- GPS 信号丢失检测机制

**涉及进程**：
1. **GPS PPS 同步与防抖进程**
   - 使用三级寄存器同步 GPS 输入信号
   - 消除亚稳态，提高系统可靠性

2. **系统主状态机进程**
   - 管理 5 种工作状态：STANDBY、CALC、TRAIN、TRAIN_DONE、ERR
   - 串口指令优先级高于自动状态跳转

3. **GPS 丢失检测进程**
   - 2 秒无 PPS 信号时判定为丢失
   - 触发 FIFO 强制刷写

**实现的具体功能**：
- ✅ 5 状态主状态机
- ✅ GPS 信号有效性检测（2 秒阈值）
- ✅ UART 指令控制接口
- ✅ 模块互联与信号路由

### 3.2 守时核心 (`timekeeping_core.v`)

**功能说明**：
- 生成本地秒脉冲 (DPPS)
- 测量 GPS PPS 与本地 DPPS 的相位差
- 接收补偿值并调整本地时钟

**涉及进程**：
1. **DPPS 生成进程**
   - 基于 25MHz 系统时钟生成 1Hz 秒脉冲
   - 支持补偿值动态调整

2. **相位差测量进程**
   - 采用差分测量法消除初始相位误差
   - 输出差值符号和数值

**实现的具体功能**：
- ✅ 本地 DPPS 生成
- ✅ 相位差测量（符号+7位数值）
- ✅ 差分测量算法
- ✅ 补偿值接收与应用

### 3.3 FIFO 写模块 (`fifo_write_module.v`)

### 3.11 长周期计数与差值运算模块 (`long_period_counter.v`)

**功能说明**：
- 实现256秒累计周期的计数
- 计算实际计数值与理想计数值的差值
- 将差值拆分为商和余数

**涉及进程**：
1. **长周期计数进程**
   - 累计256个PPS周期的时钟数
   - 计算理想计数值：25MHz × 256s = 6,400,000,000
   - 计算差值：diff_total = N_actual - N_nominal

2. **除法运算进程**
   - 将差值除以256（秒数）
   - 输出商（驯服后DPPS计数基准）
   - 输出余数（需均匀补偿的偏差量）

**实现的具体功能**：
- ✅ 256秒长周期计数
- ✅ 差值计算（N_actual - N_nominal）
- ✅ 除法运算（商+余数）
- ✅ 周期编号管理
- ✅ 差值有效标志输出

### 4.1.1 参数数据结构说明

**参数集结构**：
```
[16字节元数据] + [N字节数据区]

元数据区（0x00-0x0F）：
0x00: 魔数 (0xAA)
0x01: 参数类型 (00=单次, 01=温度段, 10=电压段, 11=综合)
0x02-0x03: 版本号（大端）
0x04-0x07: 时间戳（大端，秒数）
0x08-0x09: 数据点数（大端）
0x0A-0x0B: CRC16校验（大端）
0x0C-0x0F: 保留

数据区（0x10开始）：
实际差值数据（每个数据8位：1位符号+7位数值）
```

**地址映射表结构**：
```
位置：0x000000-0x000FFF（4KB）
每个条目：64字节
支持条目数：64个

条目结构：
0x00: 魔数 (0x55)
0x01: 参数类型
0x02-0x03: 版本号
0x04-0x07: 时间戳
0x08-0x0A: 数据起始地址（24位，大端）
0x0B-0x0D: 数据结束地址（24位，大端）
0x0E-0x0F: CRC16
0x10-0x3F: 保留
```

## 8. 硬件连接说明

### 8.1 ADC 硬件接口要求

**接口类型**：LVCMOS25 (2.5V 逻辑电平)
**工作电压**：2.5V
**参考时钟**：25MHz
**通信协议**：SPI（用于Flash通信）和UART（用于上位机通信）

### 8.2 引脚连接映射表

| 信号名称 | 物理引脚 | 方向 | 电气特性 | 功能描述 | 连接说明 |
|---------|---------|------|---------|----------|----------|
| **clk** | P21 | 输入 | LVCMOS25, PULLUP | 系统时钟（25MHz） | 连接到外部25MHz晶振 |
| **dpps_out** | P1 | 输出 | LVCMOS25, DRIVESTRENGTH=16, NONE | 补偿后的DPPS输出 | 连接到外部设备 |
| **gps_pps** | P2 | 输入 | LVCMOS25, PULLUP | GPS标准PPS信号 | 连接到北斗/GPS模块的PPS输出 |
| **rst_n** | P3 | 输入 | LVCMOS25, PULLUP | 系统复位（低有效） | 连接到复位按钮（一端接地） |
| **test_mode** | P4 | 输入 | LVCMOS25, PULLUP | 测试模式（高有效） | 连接到测试按钮（一端接地） |
| **spi_cs_n** | P5 | 输出 | LVCMOS25, DRIVESTRENGTH=16, NONE | Flash SPI片选（低有效） | 连接到Flash芯片的CS引脚 |
| **spi_miso** | P6 | 输入 | LVCMOS25, PULLUP | Flash SPI MISO | 连接到Flash芯片的MISO引脚 |
| **spi_mosi** | P11 | 输出 | LVCMOS25, DRIVESTRENGTH=16, NONE | Flash SPI MOSI | 连接到Flash芯片的MOSI引脚 |
| **spi_sclk** | P12 | 输出 | LVCMOS25, DRIVESTRENGTH=16, NONE | Flash SPI时钟 | 连接到Flash芯片的SCLK引脚 |
| **uart_rx** | P13 | 输入 | LVCMOS25, PULLUP | UART接收 | 连接到上位机的TX引脚 |
| **uart_tx** | P15 | 输出 | LVCMOS25, DRIVESTRENGTH=16, NONE | UART发送 | 连接到上位机的RX引脚 |

### 8.3 所需外部组件

| 组件类型 | 规格 | 数量 | 用途 | 连接位置 |
|---------|------|------|------|----------|
| 晶振 | 25MHz, 2.5V | 1 | 系统时钟 | P21（clk） |
| 电容 | 10pF | 2 | 晶振负载电容 | 晶振两端到地 |
| 按钮 | 轻触开关 | 2 | 复位和测试模式 | P3（rst_n）和P4（test_mode） |
| 电阻 | 10kΩ | 2 | 上拉电阻（可选，FPGA内部已配置） | 复位和测试按钮到VCC |
| Flash芯片 | W25Q64（8MB） | 1 | 数据存储 | P5-P6, P11-P12 |
| 北斗/GPS模块 | 支持PPS输出 | 1 | 时间基准 | P2（gps_pps） |
| USB转串口 | CP2102 | 1 | 上位机通信 | P13（uart_rx）和P15（uart_tx） |

### 8.4 接线步骤

**步骤1：时钟电路连接**
1. 将25MHz晶振的一端连接到P21（clk）
2. 将晶振的另一端连接到地
3. 在晶振两端各并联一个10pF电容到地

**步骤2：复位和测试电路**
1. 将复位按钮的一端连接到P3（rst_n）
2. 将测试按钮的一端连接到P4（test_mode）
3. 将两个按钮的另一端连接到地
4. 可选：在P3和P4与VCC之间各串联一个10kΩ上拉电阻

**步骤3：Flash存储连接**
1. Flash芯片CS引脚 → P5（spi_cs_n）
2. Flash芯片MISO引脚 → P6（spi_miso）
3. Flash芯片MOSI引脚 → P11（spi_mosi）
4. Flash芯片SCLK引脚 → P12（spi_sclk）
5. Flash芯片VCC → 2.5V
6. Flash芯片GND → 地

**步骤4：GPS模块连接**
1. 北斗/GPS模块PPS输出 → P2（gps_pps）
2. GPS模块VCC → 3.3V（根据模块规格）
3. GPS模块GND → 地

**步骤5：上位机通信连接**
1. USB转串口模块TX → P13（uart_rx）
2. USB转串口模块RX → P15（uart_tx）
3. USB转串口模块VCC → 5V
4. USB转串口模块GND → 地

**步骤6：DPPS输出连接**
1. P1（dpps_out） → 外部设备的时间基准输入
2. 建议使用屏蔽线缆减少干扰

### 8.5 特殊考虑因素

1. **信号完整性**：
   - 所有高速信号（如SPI时钟）应保持尽可能短的布线
   - 避免信号与电源/地线平行布线
   - 对于长距离传输，考虑使用屏蔽线缆

2. **电源稳定性**：
   - 为FPGA和Flash提供稳定的2.5V电源
   - 在电源输入处添加去耦电容（10μF和0.1μF）
   - 确保电源纹波小于50mV

3. **ESD保护**：
   - 所有外部接口应考虑ESD保护措施
   - 可使用ESD抑制二极管保护输入引脚

4. **接地设计**：
   - 采用单点接地设计
   - 确保数字地和模拟地良好连接
   - 避免地环路

5. **测试模式使用**：
   - 测试模式（test_mode=高）时，晶振稳定时间从100秒缩短到1秒
   - 仅在开发和测试时使用，实际部署时应保持低电平

6. **Flash操作**：
   - 确保Flash芯片与FPGA的电压兼容
   - 遵循SPI通信时序要求
   - 注意Flash的擦写次数限制（典型为10万次）

### 8.6 硬件调试建议

1. **时钟测试**：
   - 使用示波器验证25MHz时钟信号质量
   - 确保时钟占空比在45-55%之间

2. **GPS信号测试**：
   - 验证GPS模块PPS输出是否为1Hz脉冲
   - 检查信号幅度是否符合LVCMOS25标准

3. **Flash通信测试**：
   - 使用SPI调试工具验证Flash读写操作
   - 确认芯片ID读取正常

4. **UART通信测试**：
   - 使用串口助手发送指令测试通信
   - 验证系统状态回传功能

5. **DPPS输出测试**：
   - 使用频率计数器测量DPPS输出频率
   - 验证补偿功能是否正常工作
